# CloudFront Access-Log Historization — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Capture every request's client IP + geo/network metadata at the CloudFront edge, store it as Parquet (flat, under `raw/`) in a dedicated S3 bucket with 90-day retention. Observe-only; no query layer or dashboard built or documented.

**Architecture:** Enable CloudFront **standard logging v2** on the existing `aws_cloudfront_distribution.main`. AWS's CloudWatch-Logs "vended log delivery" subsystem (three resources: source → destination → delivery) ships logs cross-region to a new eu-central-1 S3 bucket. All three delivery resources must be created in `us-east-1` (CloudFront requirement) using the existing `aws.us_east_1` provider alias. Observe-only; no WAF, no app/Lambda/frontend changes.

**Tech Stack:** Terraform (`hashicorp/aws ~> 6.0`), AWS CloudFront, S3, CloudWatch Logs delivery API. Spec: `docs/superpowers/specs/2026-06-27-cloudfront-access-log-historization-design.md`.

## Global Constraints

- **All changes live in `packages/infrastructure/` only.** No application, Lambda, or frontend code is touched.
- **Run all Terraform with `-chdir`:** `terraform -chdir=packages/infrastructure <cmd>` (never `cd`). `init` is already done (`.terraform/` exists).
- **The three `aws_cloudwatch_log_delivery_*` resources MUST set `provider = aws.us_east_1`.** The delivery API for CloudFront only runs in us-east-1. The S3 bucket and its sub-resources use the default provider (eu-central-1).
- **Field names — already verified against the live API** (`aws logs describe-configuration-templates --service cloudfront --log-types ACCESS_LOGS`, S3 template). Use verbatim: `date, time, c-ip, c-country, asn, cs-method, cs-protocol, cs(Host), cs-uri-stem, cs-uri-query, sc-status, x-edge-result-type, x-edge-location, cs(User-Agent)`. `cs(Host)` and `cs(User-Agent)` use the parenthesized W3C form. (Parquet output and flat delivery are also confirmed allowed for the S3 destination.)
- **Bucket name:** `alexandria-cloudfront-logs-<account-id>` (matches required regex `[\w-]`).
- **Bucket conventions (match existing `s3.tf`):** `force_destroy = true`, all four public-access-block flags `true`, SSE-S3 (`AES256`).
- **Commits require explicit user approval** (repo rule: never commit/push automatically; work stays on `master`, no feature branch). Each "Commit" step below is a *proposed* commit — present the command and wait for the user's go-ahead before running it.
- **Silent-failure awareness:** if the S3 bucket policy conditions or `Resource` path are wrong, log delivery fails silently with `AccessDenied` (no error on the distribution, logs simply never appear). Task 5 verifies real delivery.

**Interfaces:**

- [ ] **Step 1: Create `logs.tf` with the bucket, caller identity, public-access block, encryption, and lifecycle**

```hcl
# Generated by Claude.
# CloudFront standard logging (v2) → dedicated S3 bucket.
# See docs/superpowers/specs/2026-06-27-cloudfront-access-log-historization-design.md

data "aws_caller_identity" "current" {}

# Dedicated bucket for CloudFront access logs (one bucket per app, by design).
resource "aws_s3_bucket" "cloudfront_logs" {
  bucket = "alexandria-cloudfront-logs-${data.aws_caller_identity.current.account_id}"
  # Allow `terraform destroy` to remove the bucket even when log objects remain.
  force_destroy = true
}

resource "aws_s3_bucket_public_access_block" "cloudfront_logs" {
  bucket                  = aws_s3_bucket.cloudfront_logs.id
  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}

resource "aws_s3_bucket_server_side_encryption_configuration" "cloudfront_logs" {
  bucket = aws_s3_bucket.cloudfront_logs.id

  rule {
    apply_server_side_encryption_by_default {
      sse_algorithm = "AES256"
    }
  }
}

# Whole-bucket 90-day expiry, no prefix filter. The bucket is dedicated to these logs
# (everything lands under raw/), so expiring all objects is correct and simplest.
resource "aws_s3_bucket_lifecycle_configuration" "cloudfront_logs" {
  bucket = aws_s3_bucket.cloudfront_logs.id

  rule {
    id     = "ExpireCloudFrontLogs"
    status = "Enabled"

    filter {} # all objects

    expiration {
      days = 90
    }

    abort_incomplete_multipart_upload {
      days_after_initiation = 3
    }
  }
}
```

## Task 2: Delivery source + bucket write policy

**Files:**
- Modify: `packages/infrastructure/logs.tf` (append)

**Interfaces:**
- Consumes: `aws_s3_bucket.cloudfront_logs.arn`, `data.aws_caller_identity.current.account_id`, and the existing `aws_cloudfront_distribution.main.arn` (defined in `cloudfront.tf`).
- Produces: `aws_cloudwatch_log_delivery_source.cloudfront` (referenced by `.name` in Task 3); `aws_s3_bucket_policy.cloudfront_logs`.

- [ ] **Step 1: Append the delivery source (us-east-1) and the bucket write policy to `logs.tf`**

```hcl
# Registers the CloudFront distribution as a log delivery source.
# MUST be created in us-east-1 (CloudFront delivery API requirement).
resource "aws_cloudwatch_log_delivery_source" "cloudfront" {
  provider     = aws.us_east_1
  name         = "alexandria-cloudfront-access-logs"
  log_type     = "ACCESS_LOGS"
  resource_arn = aws_cloudfront_distribution.main.arn
}

# Allows the log-delivery service principal to write objects into the bucket.
# Mirrors AWS's documented "AWSLogsDeliveryWrite" statement. Resource is the WHOLE
# bucket to avoid the path-prefix mismatch AWS explicitly warns about (CloudFront
# auto-prefixes the path). aws:SourceArn uses the account-wide delivery-source wildcard
# so this policy does not depend on the specific source resource.
resource "aws_s3_bucket_policy" "cloudfront_logs" {
  bucket = aws_s3_bucket.cloudfront_logs.id

  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [
      {
        Sid       = "AWSLogsDeliveryWrite"
        Effect    = "Allow"
        Principal = { Service = "delivery.logs.amazonaws.com" }
        Action    = "s3:PutObject"
        Resource  = "${aws_s3_bucket.cloudfront_logs.arn}/*"
        Condition = {
          StringEquals = {
            "s3:x-amz-acl"      = "bucket-owner-full-control"
            "aws:SourceAccount" = data.aws_caller_identity.current.account_id
          }
          ArnLike = {
            "aws:SourceArn" = "arn:aws:logs:us-east-1:${data.aws_caller_identity.current.account_id}:delivery-source:*"
          }
        }
      }
    ]
  })
}
```

## Task 3: Delivery destination + delivery (the pipe, fields, flat `raw/` layout)

**Files:**
- Modify: `packages/infrastructure/logs.tf` (append)

**Interfaces:**
- Consumes: `aws_s3_bucket.cloudfront_logs.arn`, `aws_cloudwatch_log_delivery_source.cloudfront.name`, `aws_s3_bucket_policy.cloudfront_logs` (via `depends_on`).
- Produces: `aws_cloudwatch_log_delivery_destination.cloudfront_s3`, `aws_cloudwatch_log_delivery.cloudfront` (the live logging pipeline).

- [ ] **Step 1: Append the delivery destination and delivery to `logs.tf`**

```hcl
# Where logs go + their on-disk format. Parquet output (confirmed allowed for S3).
# The "/raw" suffix on the destination ARN makes logs land flat under raw/ AND
# suppresses CloudFront's default AWSLogs/aws-account-id=<id>/CloudFront/ path.
# MUST be created in us-east-1. output_format can only be set at creation.
resource "aws_cloudwatch_log_delivery_destination" "cloudfront_s3" {
  provider                  = aws.us_east_1
  name                      = "alexandria-cloudfront-s3"
  delivery_destination_type = "S3"
  output_format             = "parquet"

  delivery_destination_configuration {
    destination_resource_arn = "${aws_s3_bucket.cloudfront_logs.arn}/raw"
  }
}

# The pipe: links source → destination and selects fields. No s3_delivery_configuration
# block => flat delivery under the raw/ prefix (no Hive partitioning, no suffix_path).
# depends_on the bucket policy: CreateDelivery validates write access, so the policy
# must exist first or delivery creation fails.
resource "aws_cloudwatch_log_delivery" "cloudfront" {
  provider                 = aws.us_east_1
  delivery_source_name     = aws_cloudwatch_log_delivery_source.cloudfront.name
  delivery_destination_arn = aws_cloudwatch_log_delivery_destination.cloudfront_s3.arn

  # Field names verified against the live config-templates API. cs(Host)/cs(User-Agent)
  # are parenthesized; c-country + asn give per-row geo/network from the viewer IP.
  record_fields = [
    "date",
    "time",
    "c-ip",
    "c-country",
    "asn",
    "cs-method",
    "cs-protocol",
    "cs(Host)",
    "cs-uri-stem",
    "cs-uri-query",
    "sc-status",
    "x-edge-result-type",
    "x-edge-location",
    "cs(User-Agent)",
  ]

  depends_on = [aws_s3_bucket_policy.cloudfront_logs]
}
```

Every request to `alexandria.isnan.eu` is logged at the CloudFront edge and delivered as
**Parquet** to the dedicated bucket `alexandria-cloudfront-logs-<account-id>` (eu-central-1), retained

**Interfaces:**

- Produces: confirmed log objects in S3 with a populated `c-ip` / `c-country`.

Expected: prompts with the adds from Tasks 1–3 (bucket + 4 sub-resources, source, policy, destination, delivery). Type `yes`. Apply completes without error. (Get user approval before applying — this provisions live AWS resources.)

- [ ] **Step 2: Confirm logging shows as enabled on the distribution**

Run: `aws logs get-delivery-source --name alexandria-cloudfront-access-logs --region us-east-1`
Expected: JSON with `"service": "cloudfront"`, `"logType": "ACCESS_LOGS"`, and the distribution ARN under `resourceArns`.

- [ ] **Step 3: Generate traffic**

Run: `for i in $(seq 1 10); do curl -s -o /dev/null https://alexandria.isnan.eu/; done`
Expected: command returns (no output). This seeds requests to be logged.

- [ ] **Step 4: Wait, then check objects landed (delivery latency is minutes, not seconds)**

Run (re-run until objects appear; allow up to ~15 min):
`aws s3 ls s3://alexandria-cloudfront-logs-<account-id>/raw/ --recursive | head`
Expected: at least one `.parquet` object directly under `raw/`.
**If still empty after ~15 min:** the bucket policy is the prime suspect (conditions or `Resource` path) — re-check Task 2 against the AWS `AWSLogsDeliveryWrite` statement, and confirm the delivered path is covered by the policy `Resource` (`/*` covers `raw/*`).

- [ ] **Step 5: Confirm the client IP and country are actually populated**

Download one object and inspect it (any Parquet reader; example with DuckDB if available):
Run: `aws s3 cp s3://alexandria-cloudfront-logs-<account-id>/$(aws s3 ls s3://alexandria-cloudfront-logs-<account-id>/raw/ --recursive | awk '{print $4}' | grep '.parquet$' | head -1) /tmp/cf.parquet`
Then: `duckdb -c "SELECT * FROM read_parquet('/tmp/cf.parquet') LIMIT 3;"` (or `parquet-tools cat /tmp/cf.parquet | head`).
Expected: rows with a non-empty client-IP column and a 2-letter country code.
